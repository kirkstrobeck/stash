- Code, types, all implementation must be highly strict
- Never use `else` or `else if`
- Make booleans optional
- Defer to defaults where possible
- Avoid arity, prefer an object literal parameter
- Never mutate
- Never use `null`
- Code files must not exceed 150 lines
- Every file must be stand-alone and authoritative
- Avoid classes and complexity
- All UI must run at 60 FPS or greater
- Default to sentence-case for text

JS/TS-specific
- Never use `let` or `push`
- Never use `enum`
- Avoid `as` and `any`
- Object literals with more than one key must be multiple lines and alpha-sorted
